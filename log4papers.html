摘要

问题: 现有 CTR 预测模型 (DeepFM, DCN) 对多字段 + 序列特征的建模能力不足。

方法: 提出 TokenFormer，将 CTR 预测重构为统一 Token 流上的序列建模任务：X = [F | SEP | T | SEP | V]。引入 BFTS (Bidirectional Field-aware Token Selection) 注意力、NLIR (Non-Linear Information Routing) 门控、Inter-layer Residuals 和 SWA (Sliding Window Attention)。

结果: 在 Avazu 数据集 (40M 样本) 上，TokenFormer 达到 AUC=0.7463，较 PCVRHyFormer 基线 (0.7448) 提升 +0.0015。消融实验表明 Inter-layer Residuals (+0.0120) 和 SWA (+0.0039) 是核心增益来源，而 MoE、Global Token、Per-Field Tokenization 贡献微弱 (<0.0010)。基于此推出 TokenFormer-Lite，参数量减少 42% (3.4M vs 5.9M)，性能持平。

1. 引言

1.1 动机

传统 CTR 模型 (DeepFM, xDeepFM, DCN) 将特征分为"静态字段"(user/item 属性)和"动态序列"(行为历史)分别处理，导致：

  • 信息隔离: 静态特征和动态序列的交互受限
  • 扩展性差: 新增字段需重新设计特征交叉模块
  • 参数量大: 为每类特征设计独立编码器

1.2 核心思想

受 NLP 中 "将所有内容视为 Token" 的启发，TokenFormer 将 CTR 预测中的所有信息统一为 Token 流：

X = [F | <SEP> | T | <SEP> | V]

其中 F=字段 Token (user/item 属性), T=序列 Token (行为历史), V=目标 Token (预测目标)。所有 Token 通过统一的 Transformer 层交互，实现：

  • 统一交互: F-T, F-V, T-V 的任意两两交互
  • 即插即用: 新增字段 = 新增 Token，无需改架构
  • 参数共享: 统一注意力机制处理所有 Token

2.2 BFTS 注意力机制

Bidirectional Field-aware Token Selection (BFTS) 在标准因果注意力基础上增加两条规则：

  1. F-F 双向: 所有 F-Token 之间双向可见 (字段间全交互)
  2. F-丢弃: T/V Token 不 attend 到 F-Token (避免信息泄露)

注意力掩码:

visible = causal & (window_size) & (~discard_F)

2.3 NLIR 门控

Non-Linear Information Routing (NLIR) 通过门控机制控制信息流：

G = sigmoid(X W_g)  # 信息门
I_tilde = G * Attn(X)  # 选择性保留
I = X + I_tilde  # 残差连接

2.4 Inter-layer Residuals

每 2 层保存中间输出，在后续层相加：

if layer_idx % 2 == 0: residual_stack.append(X)
if layer_idx % 2 == 1: X = X + residual_stack.pop()

2.5 SWA (Sliding Window Attention)

上层使用滑动窗口注意力降低复杂度：

window_size = [64, 32, 16]  # 逐层递减

复杂度从 O(L²) 降至 O(L × window_size)。

5. TokenFormer-Lite

5.1 设计动机

消融实验表明，MoE (+0.0010)、Global Token (+0.0005)、Per-Field (+0.0007) 的贡献均微弱，但显著增加了参数量和实现复杂度。TokenFormer-Lite 仅保留核心组件：

  • 统一 Token 流 + BFTS 注意力
  • Inter-layer Residuals (+0.0120)
  • SWA (+0.0039)
  • Down-matrix 小初始化 (P0)

6. 结论

主要贡献

  1. 统一 Token 流范式: 将 CTR 预测重构为序列建模任务，实现静态特征和动态序列的统一交互
  2. BFTS 注意力: 针对 CTR 场景设计双向字段感知注意力，支持字段间全交互
  3. Inter-layer Residuals: 跨层残差连接显著提升深层模型性能 (+0.0120 AUC)
  4. TokenFormer-Lite: 通过消融实验指导的精简架构，参数量减少 42%，性能持平

局限与未来工作

  • 当前仅在 Avazu 数据集验证，需在 Criteo、Amazon 等数据集进一步验证
  • MoE 在大规模数据上的效果待验证 (当前小数据集过拟合)
  • 对比学习 (P6) 的实现可进一步优化 (当前为简化版)
