2. 注意力机制优化

2.1 Grouped Query Attention (GQA)

原理

将多头注意力中的 K/V 头数减少为 Q 头数的 1/n，在保持表达能力的同时显著降低 KV 缓存和参数量。对于 PCVRHyFormer 的短序列场景，GQA 可以减少约 20-30% 的注意力参数量。

适用性分析

当前框架的 scaled_dot_product_attention 支持任意 num_heads，只需调整 K/V 的投影维度。适合 Avazu/Criteo 的轻量场景。

实施建议：添加 num_kv_heads 参数，令 num_kv_heads = num_heads // group_size。K/V 投影从 (d_model, d_model) 变为 (d_model, num_kv_heads * head_dim)。
高优先级 新组件 预期 AUC 持平 | 参数量 -15%

2.2 门控多头注意力 (Gated MHA)

原理

在 attention 输出后引入一个可学习的门控：

gate = sigmoid(Linear(x))
output = gate * attn_output + (1 - gate) * x

允许模型自适应地决定何时依赖注意力、何时保留原始特征。

适用性分析

实验发现 rank_mixer_mode=none 最优，说明模型倾向于"少干预"。门控机制赋予模型选择权，可能比强制 rank mixing 更有效。

中优先级 新组件 预期 AUC +0.1~0.3%

2.3 线性注意力 / 稀疏注意力

原理

将 softmax(QK^T)V 替换为核技巧近似 (phi(Q) phi(K)^T V)，将复杂度从 O(n^2) 降至 O(n)。适合超长序列（如 Amazon 的 max_len=50+）。

适用性分析

Avazu 序列较短（top_k=50），O(n^2) 开销不大。但对于未来可能的长序列场景或更大的 batch，线性注意力可显著加速。框架中的 flash_attention 已支持高效实现，可作为回退。

低优先级 新组件 预期 训练速度 +20% | AUC 持平

3. Block 结构优化

3.1 Pre-LN + RMSNorm 替代 LayerNorm

现状

框架已内置 RMSNorm（支持 torch/tilelang/triton 后端），但当前模型可能使用 LayerNorm。实验显示 rank_mixer_mode=none 时使用了 LayerNorm。

优化方向

1. 将所有 LayerNorm 替换为 RMSNorm（减少 bias 参数，提升稳定性）
2. 采用 Pre-LN 结构（归一化在残差连接之前），缓解深层网络梯度问题，使 num_blocks>2 时性能不再下降。

框架就绪度：100% — RMSNorm 已在 taac2026/infrastructure/modeling/normalization.py 中完整实现，仅需在模型中替换调用。
高优先级 框架已支持 预期 AUC +0.1~0.2% | 训练稳定性提升

3.2 SwiGLU FFN

原理

SwiGLU 是 LLaMA、PaLM 等模型采用的 FFN 变体：

SwiGLU(x) = (W1*x) * sigmoid(W2*x) @ W3

相比标准 FFN (W1*x @ W2)，SwiGLU 引入门控非线性，表达能力更强。

实验关联

E1 中 SwiGLU encoder 表现不佳 (0.7199 vs 0.7354)，但 encoder 和 FFN 职责不同。当前 tokenizer 的 project 已使用 SiLU，说明框架兼容此类激活函数。

注意：SwiGLU 使用 3 个权重矩阵而非 2 个，参数量增加 50%。建议将 hidden_mult 从 4 降至 2-3 以保持总参数量不变。
中优先级 新组件 预期 AUC +0.1~0.3%

3.3 并行 Transformer Block (ParFormer-style)

原理

标准 Transformer Block 串行执行 Attention -> FFN。并行块同时计算两者后相加：

y = x + Attention(LN(x)) + FFN(LN(x))

减少层间通信，加速训练；且 Pre-LN 下理论等价于标准结构。

适用性

对 num_blocks=2 的浅层网络收益有限，但对未来扩展到更深网络（如 Criteo 的 3.29M 参数已验证 b=8）可缓解通信瓶颈。

低优先级 新组件 预期 训练速度 +5~10%

3.4 Stochastic Depth (DropPath)

原理

训练时以概率 p 随机"跳过"整个 block（将其输出设为输入），测试时使用所有 block。作为深层网络的正则化手段。

关键关联

E6 显示 b=4/8 性能低于 b=2，正是过拟合信号。Stochastic Depth 可使 b=4/8 在正则化下恢复性能，突破当前深度瓶颈。

核心洞察：这是解决"倒U型深度曲线"最直接的手段。对 b=4 设置 drop_path_rate=0.1~0.2，可能使 b=4 达到或超过 b=2 的性能。
高优先级 新组件 预期 b=4 AUC 从 0.7358 提升至 0.7370+

4. 特征交互优化

4.1 显式二阶特征交叉层 (Explicit FM Layer)

原理

在 NS tokenizer 输出后，添加显式的特征交叉：

# 输入: (batch, T, d_model)
cross = sum_i sum_j (x_i * x_j)  # 二阶交互
output = Linear(concat([x, cross]))

CTR 预测中特征交叉至关重要（如 "device_type x hour" 的组合效应）。

关联分析

rank_mixer_mode=none 最优说明模型在"隐式"地学习交叉，但显式引入可能更数据高效。可替代被废弃的 rank_mixer。

高优先级 新组件 预期 AUC +0.2~0.5%

4.2 特征级自注意力 (Feature-wise Self-Attention)

原理

在 NS token 之间（即不同特征组之间）应用轻量级 self-attention：

# NS tokens: (batch, num_ns, d_model)
# 使用单头或低秩 attention
attn_output = Attention(NS_tokens, NS_tokens, NS_tokens)

让不同特征组之间自适应地交换信息，替代硬编码的 rank mixing。

优势

相比 rank_mixer 的固定模式，自注意力让数据决定交互权重。num_ns 很小（当前为 3），attention 计算开销可忽略。

中优先级 新组件 预期 AUC +0.1~0.3%

4.3 FiBiNET-style Squeeze-and-Excitation

原理

为每个特征 token 学习一个重要性权重：

squeeze = mean(tokens, dim=-1)      # (batch, T)
excite = Linear2(SiLU(Linear1(squeeze)))  # (batch, T)
weights = sigmoid(excite)
output = tokens * weights.unsqueeze(-1)

适用性

CTR 模型中不同特征重要性差异巨大（如 user_id vs 次要特征）。SENET 让模型自动抑制噪声特征，增强信号特征。

中优先级 新组件 预期 AUC +0.1~0.2%

5. 训练算法优化

5.1 Muon 优化器

现状

框架已内置 Muon 优化器（taac2026/infrastructure/optimization/muon.py），对 >=2D 参数使用 Newton-Schulz 正交化更新，对向量参数回退到 AdamW。

优化建议

当前使用 AdamW。Muon 在矩阵参数上通过正交约束实现更稳定的梯度更新，对 attention/FFN 的权重矩阵尤其有效。建议在 Criteo/Amazon 大数据集上优先尝试。

框架就绪度：100% — 只需将 dense_optimizer_type 从 "adamw" 改为 "muon"。
高优先级 框架已支持 预期 AUC +0.1~0.3% | 训练更稳定

5.2 Cosine LR Schedule + Warmup

现状

框架已支持 cosine/linear scheduler + warmup（taac2026/infrastructure/optimization/schedules.py），但当前配置使用 "none"。

优化建议

实验训练 5000 steps，使用 cosine decay（min_lr_ratio=0.1）+ 200-step warmup 可让模型在训练后期更精细地收敛，避免震荡。

框架就绪度：100% — 设置 scheduler_type="cosine", warmup_steps=200, min_lr_ratio=0.1。
中优先级 框架已支持 预期 LogLoss -0.001~0.003

5.3 EMA (Exponential Moving Average)

现状

框架已完整支持 EMA（PCVREMAConfig），但当前未启用。EMA 在验证和推理时使用参数的移动平均，平滑训练噪声。

优化建议

启用 EMA：decay=0.999, start_step=1000。特别适合当前"轻量模型+固定数据"场景，可挤出最后一点性能。

框架就绪度：100% — 设置 ema_enabled=true, ema_decay=0.999, ema_start_step=1000。
中优先级 框架已支持 预期 AUC +0.05~0.15%

5.4 标签平滑 (Label Smoothing)

原理

将 hard label [0, 1] 替换为 soft label [0.05, 0.95]，防止模型对训练样本过度自信，提升泛化。

CTR 场景

CTR 数据中存在大量噪声标签（误点击、机器人流量），标签平滑可缓解标签噪声的影响。

中优先级 需修改 loss 计算 预期 AUC +0.05~0.1%

5.5 数据增强

框架支持

框架已内置多种数据增强（PCVRDataPipelineConfig）：

  • feature_mask: 随机 mask 非序列特征
  • domain_dropout: 随机丢弃整个序列域
  • sequence_crop: 随机截取序列子窗口
  • nonseq_sparse_dropout: 稀疏特征 dropout

优化建议

当前均未启用。建议对 Avazu 启用：
- feature_mask probability=0.1
- nonseq_sparse_dropout probability=0.05
对 Amazon 启用 sequence_crop (views_per_row=2) 增加序列多样性。

框架就绪度：100% — 在配置中启用对应 transform 即可。
中优先级 框架已支持 预期 AUC +0.05~0.2%

6. 序列编码优化

6.1 多尺度序列编码

原理

使用多个并行的序列编码器，分别处理不同时间粒度的序列：

encoder_recent   -> 最近 10 个交互
encoder_medium   -> 最近 50 个交互
encoder_long     -> 完整序列

最后将多尺度表示拼接或注意力融合。

适用性

Amazon 数据集上序列长度 50，用户行为存在明显的短期/长期兴趣分离。多尺度编码可分别捕捉"最近浏览"和"长期偏好"。

中优先级 新组件 Amazon 预期 AUC +0.3~0.5%

6.2 T5-style 相对位置偏置

原理

RoPE 实验显示无收益。替代方案：使用可学习的相对位置偏置矩阵 B，直接加到 attention score：

attention_score = QK^T / sqrt(d) + B[pos_i - pos_j]

比绝对位置编码更短序列友好。

优势

T5 的相对偏置在短文本任务上表现优异。对 Avazu 的 device_id 序列（长度<50），相对偏置比 RoPE 更自然地建模"相邻交互"关系。

中优先级 新组件 预期 AUC +0.05~0.15%

6.3 序列统计特征注入

现状

框架的 ModelInput 已包含 seq_stats（每个序列域 6 维统计量），但模型可能未充分利用。

优化建议

将 seq_stats（如序列长度、均值、方差等）通过 MLP 投影后与序列编码输出拼接，作为额外的全局上下文信号。

框架就绪度：100% — 数据已传入，只需在模型中接入。
低优先级 框架已支持 预期 AUC +0.05%

9.2 Stochastic Depth (DropPath) 伪代码

"""在 PCVRHyFormer 的 block forward 中嵌入"""
import torch.nn as nn

class PCVRBlock(nn.Module):
    def __init__(self, d_model, num_heads, drop_path_rate=0.0):
        super().__init__()
        self.attn = MultiHeadAttention(d_model, num_heads)
        self.ffn = FeedForward(d_model)
        self.norm1 = RMSNorm(d_model)
        self.norm2 = RMSNorm(d_model)
        self.drop_path_rate = drop_path_rate

    def forward(self, x, mask=None):
        # Pre-LN: 归一化在子层之前
        # Stochastic Depth: 训练时随机跳过整个子层
        if self.training and torch.rand(1).item() < self.drop_path_rate:
            attn_out = 0
        else:
            attn_out = self.attn(self.norm1(x), mask)
        x = x + attn_out

        if self.training and torch.rand(1).item() < self.drop_path_rate:
            ffn_out = 0
        else:
            ffn_out = self.ffn(self.norm2(x))
        x = x + ffn_out
        return x

9.3 显式二阶 FM 层伪代码

"""替代 rank_mixer 的显式特征交叉模块"""
import torch
import torch.nn as nn

class ExplicitFeatureCross(nn.Module):
    def __init__(self, num_tokens, d_model, cross_dim=64):
        super().__init__()
        self.num_tokens = num_tokens
        self.cross_proj = nn.Linear(d_model, cross_dim)
        # 输出投影: 原始特征 + 交叉特征
        self.output_proj = nn.Sequential(
            nn.LayerNorm(num_tokens * d_model + cross_dim),
            nn.Linear(num_tokens * d_model + cross_dim, num_tokens * d_model),
            nn.SiLU(),
            nn.LayerNorm(num_tokens * d_model),
        )

    def forward(self, tokens):
        # tokens: (batch, num_tokens, d_model)
        B, T, D = tokens.shape
        # 降维后的特征用于交叉
        z = self.cross_proj(tokens)  # (B, T, cross_dim)
        # 二阶交叉: sum_{i<j} z_i * z_j
        square_sum = (z ** 2).sum(dim=1)           # (B, cross_dim)
        sum_square = z.sum(dim=1) ** 2              # (B, cross_dim)
        cross = 0.5 * (sum_square - square_sum)     # (B, cross_dim)
        # 拼接原始特征和交叉特征
        flat = tokens.view(B, T * D)
        combined = torch.cat([flat, cross], dim=-1)
        output = self.output_proj(combined).view(B, T, D)
        return output

9.4 SENET 通道注意力伪代码

"""Squeeze-and-Excitation for feature tokens"""
class SENET(nn.Module):
    def __init__(self, num_tokens, reduction=2):
        super().__init__()
        hidden = max(1, num_tokens // reduction)
        self.fc = nn.Sequential(
            nn.Linear(num_tokens, hidden),
            nn.SiLU(),
            nn.Linear(hidden, num_tokens),
            nn.Sigmoid(),
        )

    def forward(self, tokens):
        # tokens: (batch, num_tokens, d_model)
        # Squeeze: 沿 d_model 维度取平均
        squeeze = tokens.mean(dim=-1)       # (batch, num_tokens)
        # Excitation: 学习每个 token 的重要性
        excite = self.fc(squeeze)            # (batch, num_tokens)
        # Scale
        return tokens * excite.unsqueeze(-1) # (batch, num_tokens, d_model)
