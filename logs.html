Phase 4: TokenFormer 集成与训练

TokenFormer 简介

TokenFormer 是腾讯提出的统一多字段+序列推荐模型，在 sample_1000_raw 数据集上达到 AUC = 0.849059。核心创新：

  • 统一 Token 流: X = [F | <SEP> | T | <SEP> | V]
  • Per-Field Tokenization: 每个特征一个 token，带 ResSwiGLU 残差
  • BFTS 注意力: 底层 full causal + 上层 sliding window attention
  • NLIR Gating: sigmoid 门控注意力
  • RMSNorm + SwiGLU FFN: LLM 风格 Pre-Norm 架构

Phase 4: TokenMixer-Large 论文学习

论文信息

属性内容
标题TokenMixer-Large: Scaling Up Large Ranking Models in Industrial Recommenders
作者Yuchen Jiang, Jie Zhu, Xintian Han, et al. (ByteDance AML)
arXiv2602.06563v2 [cs.IR]
发表ACM 2026
规模线上 7B 参数，线下 15B 参数

核心贡献

1. Mixing & Reverting 操作
解决 RankMixer/TokenMixer 的维度不匹配问题。原始 TokenMixer 将 T 个 token 混合为 H 个新 token，导致残差连接无法直接传播。
TokenMixer-Large 设计对称的 "混合-恢复" 结构：第一层混合信息，第二层恢复维度，确保输入输出维度一致。

2. Inter-layer Residuals + Auxiliary Loss
解决深层模型梯度消失问题：
- 间隔残差连接：每 N 层添加跨层残差
- 辅助损失：中间层输出也参与损失计算
- Down-matrix 小初始化：降维矩阵使用小值初始化，稳定深层训练

3. Sparse Per-token MoE
升级 RankMixer 的 ReLU-MoE 为 Sparse Per-token MoE：
- "Sparse Train, Sparse Infer" 统一范式（训练和推理都稀疏）
- 每 token 独立选择专家，避免动态激活问题
- 配合 FP8 量化和 Token Parallel，大幅降低训练/推理成本

4. Pure Model 哲学
移除碎片化算子（LHUC, DCNv2 等），仅用堆叠的 TokenMixer Block。
广告主干网络 MFU 提升至 60%。

与当前 TokenFormer 对比

维度TokenMixer-Large当前 TokenFormer差距
残差设计Mixing + Reverting，维度一致标准 Pre-Norm 残差可借鉴
深度扩展Inter-layer residuals + Aux loss无特殊设计可借鉴
MoESparse Per-token MoE无 MoE可添加
Global Token[CLS] 风格全局 token无全局 token可添加
架构纯度Pure model，无碎片化算子相对纯净已接近

可借鉴的优化方案

方案 1: Global Token (全局信息聚合)

# 在 TokenFormer 输入中添加 Global Token
X_G = MLP_g(concat[G_1, ..., G_T])  # 聚合所有组信息
X = concat[X_G, X_0, ..., X_T]      # 全局 token + 原始 tokens
预期收益: +0.001-0.003 AUC
实现复杂度: 低（添加一个 MLP + concat）
参数量增加: ~d_model^2 (可忽略)

方案 2: Inter-layer Residuals (间隔残差)

# 每 N 层添加跨层残差连接
if layer_idx % interval == 0 and layer_idx > 0:
    X = X + residual_stack.pop()  # 跨层残差
预期收益: +0.002-0.005 AUC（深层模型时）
实现复杂度: 中
适用条件: num_blocks >= 6 时效果显著

方案 3: Auxiliary Loss (辅助损失)

# 中间层输出也计算 BCE loss，加权求和
loss = bce(final_output) + lambda * sum(bce(mid_output) for mid in mid_layers)
预期收益: +0.001-0.004 AUC
实现复杂度: 高（需修改训练框架）
风险: 可能干扰主任务收敛

方案 4: Sparse Per-token MoE (稀疏专家)

# 替换 FFN 为 Sparse MoE
# 每 token 选择 top-k 专家
expert_weights, expert_indices = router(token)
output = sum(w * expert_i(token) for w, i in zip(expert_weights, expert_indices))
预期收益: +0.003-0.008 AUC
实现复杂度: 高
参数量: 可增加 2-4x 但计算量不变

方案 5: Down-matrix Small Initialization

# 降维矩阵使用小值初始化
nn.init.normal_(W_down, std=0.02 / sqrt(depth))
预期收益: 训练稳定性提升，深层模型收敛更快
实现复杂度: 极低（一行代码）
适用条件: num_blocks >= 6
